## The thirty-second pitch

[Ollama](https://ollama.com) is a free, open-source program that runs an AI model on your computer. You install it, you pull a model (one command), and you can chat with that model — or have other software call it — without anything ever leaving your machine.

That's it. The model lives on your hard drive. The conversation happens on your laptop's processor or GPU. There is no cloud subscription, no API key, no usage meter, and nobody on the other side of an internet connection writing your prompts to a log file.

For a small business, that combination — capable AI, zero recurring cost, total data privacy — is rare enough that it's worth understanding even if you decide it isn't right for you today.

## When a small business should actually use it

Three honest use cases we put in front of clients:

**1. Any task involving regulated client data.**
HIPAA-covered patient information, GLBA-covered financial details, attorney-client privileged material, employment investigation notes. Anything where putting the data into ChatGPT or Claude requires a Data Processing Addendum, an acceptable-use policy, and a cyber-insurance question. Putting it into a local Ollama model requires none of those, because the data never leaves the machine.

**2. Workflows that would be expensive at the API rate.**
Bulk-summarizing a backlog of voicemail transcripts. Re-categorizing five years of email. Translating internal documentation into Spanish for a bilingual office. The kind of one-time large-volume jobs where the per-token cost of a frontier API would actually add up. The local model is slower per request, but the per-request cost is zero.

**3. Offline reliability.**
Construction company in Lakewood Ranch, field office in a building with bad cell signal and unreliable wifi, foreman wants to draft an estimate using AI on a tablet at the job site. A cloud AI is unusable. An Ollama-equipped laptop in the truck still works.

## When to skip it

Be honest about this too. Ollama is not the right tool for:

- **Daily-driver chat for non-technical staff.** The user experience is rougher than ChatGPT or Claude. Your bookkeeper is going to find it more annoying.
- **Tasks where you want frontier-quality output.** The best open models in 2026 are excellent, but Claude Opus and GPT-5 are still meaningfully sharper at hard reasoning, complex code, and nuanced writing. For drafting a tricky client email, use a frontier API.
- **Office laptops without a real GPU.** Ollama runs CPU-only, but the experience is painful. A 2024-or-newer Apple Silicon Mac, a Windows laptop with at least a 16 GB GPU, or a workstation with an NVIDIA card from the last three years is the realistic floor.

## The ten-minute install

This is the entire setup, end to end:

1. Download Ollama from [ollama.com/download](https://ollama.com/download). Mac, Windows, and Linux installers are all on the same page. (~3 min including download.)

2. Open the new app. On Mac it lives in the menu bar. On Windows it lives in the system tray. There is no UI to click through — Ollama is a background server.

3. Open a terminal (Terminal on Mac, PowerShell on Windows). Run:

   ```
   ollama pull llama3.2
   ```

   This downloads the Llama 3.2 model — about 2 GB — to your machine. Replace `llama3.2` with `hermes4`, `mistral`, `qwen3`, or any other model from [ollama.com/library](https://ollama.com/library). (~5 min on a normal home internet connection.)

4. In the same terminal, run:

   ```
   ollama run llama3.2
   ```

   You're now chatting with the model. (~2 min to verify it works.)

That's the whole install. From here you can either keep using the terminal, install a UI like [Open WebUI](https://openwebui.com) on top, or wire Ollama into a custom integration via its built-in HTTP API (it listens on `localhost:11434` by default).

## Which model to pull

The library is large and grows weekly. The picks we usually start clients with:

- **`llama3.2:3b`** — Meta's small Llama. 3 billion parameters, fits easily on a laptop, fast. Good for summarization and short writing tasks. The "everyone should have this even if they pull other models" baseline.
- **`hermes4`** — The Nous Research [Hermes line](/blog/hermes-ai-uncensored-open-model-business-2026) — strips the corporate-assistant tuning, gives you direct answers. Right pick for legal, medical, and any work involving sensitive subject matter that the corporate models flinch at.
- **`mistral-small`** — Mistral's small workhorse. European-trained, strong on French and other European languages, surprisingly good at structured output. Right pick for any office that does real work in a non-English language.
- **`qwen3:14b`** — Alibaba's latest. Strong on coding and reasoning. The right pick if you have a workstation-class GPU and want the closest-to-frontier open model available.

Pull one. If it doesn't fit your need, pull another. They all live alongside each other on disk; switching is one terminal command.

## The integration story

Ollama exposes a simple HTTP API on `localhost:11434` that mimics the OpenAI API format. That means almost any tool that knows how to talk to OpenAI can be pointed at Ollama with a single config change. Useful integrations we use with clients:

- **Continue, Cursor, Claude Code** — all support Ollama as a provider. Your developer can do AI-assisted coding without sending source code to a vendor.
- **Open WebUI** — a free chat UI that looks like ChatGPT and runs against Ollama. Drop this on a small server and your whole office has a private chat tool.
- **n8n / Make.com** — both have Ollama integration nodes for automating workflows.
- **Our own [OpenClaw](/blog/openclaw-local-ai-agent-runtime-2026) runtime** — supports Ollama as a back-end so the same agents that run against Claude can also run against a local model when the workflow needs it.
